 🧠 Arquitectura MVP 🧱
📌 Objetivo: Organizar la app usando el patrón Model-View-Presenter
(MVP) para lograr mayor claridad, modularidad y mantenibilidad.

🧩 División:
  ●​ View: Fragmentos y pantallas que ve el usuario 👁️
  ●​ Model: Lógica de negocio (lista de productos, carrito)

  ●​ Presenter: Conecta la lógica con la interfaz 👨‍🏫​


✅ Ventajas del MVP:
  ●​ 🔬 Permite pruebas unitarias fácilmente
  ●​ 🔄 Reutilización de código
  ●​ 📏 Mejora la escalabilidad y organización del proyecto​
